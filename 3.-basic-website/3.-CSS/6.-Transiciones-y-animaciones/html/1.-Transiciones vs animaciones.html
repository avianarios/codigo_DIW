Transiciones vs Animaciones en CSS

¿Qué son las Transiciones?

Las transiciones permiten cambiar el valor de una propiedad de un elemento en respuesta a un evento.

Son ideales para efectos simples

No se ejecutan por sí solas; necesitan un "disparador" que puede ser un cambio de estado o evento (como hover, focus, o la modificación de una propiedad en JavaScript).

Son más simples que las animaciones y están limitadas a un cambio entre dos estados (el inicial y el final). No permiten control detallado sobre los pasos intermedios del cambio.

Se definen con la propiedad transition.

Se ejecutan una sola vez cuando se dispara el cambio de estado.

Ejemplo de Transición

Pasa el ratón por aquí


.caja-transicion{
    text-align: center;
    padding: 20px;
    margin: 10px auto;
    width: 200px;
    color: white;
    font-weight: bold;
    border-radius: 5px;
    background-color: blue;
}

.transicion-color-fondo {
    transition: background-color 1s;
}

.caja-transicion:hover {
    background-color: red;
}

Ejemplo de transición controlado mediante JavaScript

Las transiciones también pueden ejecutarse mediante el cambio dinámico de clases. En este caso, el "disparador" es el cambio en el valor de una propiedad transicionada.

Ejemplo


button {
    padding: 10px 20px;
    background-color: #4CAF50;
    color: white;
    border: none;
    border-radius: 5px;
    cursor: pointer;
}                

.transicion-color-fondo {
    transition: background-color 1s;
}
        
.fondo-rojo {
    background-color: red;
}

JavaScript:


let cambio = document.getElementById("boton_color");
cambio.addEventListener("click", () => cambio.classList.toggle("fondo-rojo"));

¿Qué son las Animaciones?

Las animaciones permiten definir cambios complejos con múltiples estados usando @keyframes

Son ideales para secuencias repetitivas o automáticas.

Pueden ejecutarse automáticamente (cuando se carga la página) o controlarse mediante JavaScript.

Ofrecen mayor control y permiten definir múltiples estados (keyframes) para crear secuencias complejas.

Requieren la propiedad animation y el uso de @keyframes para definir los pasos.

Pueden configurarse para repetirse (infinite) o limitar el número de iteraciones con animation-iteration-count.

Ejemplo de Animación automática

Esto es una animación

.caja-animacion {
                width: 100px;
                height: 100px;
                background-color: blue;
                animation: cambio-color 2s infinite;
            }

            @keyframes cambio-color {
                0% { background-color: blue; }
                33% { background-color: green; }
                66% { background-color: red; }
                100% { background-color: blue; }
            }

Diferencias Principales

Característica Transiciones Animaciones
Desencadenante Cambio de estado/evento Automáticas o cambio de estado/evento
Complejidad Limitadas a dos estados Secuencias complejas con keyframes
Ciclos Una sola vez por cambio Pueden repetirse (infinite)
Definición Usan transition Usan @keyframes y animation
Flexibilidad Menor Mayor
