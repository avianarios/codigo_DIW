Flexbox properties acting over flex items

align-self

Align-self allows to define behaviour only for one element in the traverse axis (i.e. the axis perpendicular to the main axis of the flex container). Some values require some empty space for the elements to expand. Values: flex-start, flex-end, center, baseline y stretch

align-self

The base alignment is align-item:center. If not specified, the flex boxes takes all the available space

Third element aligns at the end (align-self:end), 7th at stretch, 8th at start

Grow properties

flex-basis, flex-grow y flex-shrink

flex: a b c is a shortcut for the three properties. If a=number -> equivalent to flex-grow, if a=size -> equivalent to flex-basis if a and b are set, a=grow and b=shrink if it is a number or b=flex-basis if it is a size if a, b and c, then a=grow, b=shrink and c=basis

flex-basis

Defines the initial size of the flexbox item on its MAIN AXIS. It may grow due to other properties before being presented to user

If not defined: flex boxes take the minimum space to accomodate their content

If defined: flex boxes are rendered with this size unless not enough room on the container. Anyway, proportions are respected

Boxes 3 and 6 have changed theis size. Wrap is allowed

flex-grow

It controls how FREE SPACE is assigned to flex elements within the flex container. Free space is the space left AFTER all flex items have been renderized

Flex-grow will not force items to wrap. If there's no free space, no space is redistributed

If there's only one element with flex-grow, it'll take all free space. If there's more than one, free space will be distributed among them according to flex-grow

item 2 has flex-grown:2 and item 4, flex-grown 3. Free space will be splitted into 5 parts, being given 2 to item 2 and 3 to item 4

flex-shrink

The opposite of flex-grow: it determines how much space is taken away from flex items when the container reduces its size.

If flex-shrink is undefined, it behaves as if it were set to flex-shrink: 1.

When a container starts reducing its size, all flex items shrink proportionally to their flex-shrink values.

Items with flex-shrink: 0 stop shrinking when they reach their base size.

If all items have flex-shrink: 0 and the container continues to shrink, items will wrap to the next line (if wrapping is enabled) or overflow the container (if wrapping is disabled).

item 5 has flex-shrink:2 and item 7, flex-shrink: 3
