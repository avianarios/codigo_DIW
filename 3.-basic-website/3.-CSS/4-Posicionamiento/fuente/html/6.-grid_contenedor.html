Grid basic therminology

Flex gave an illusion of having multiline by using wrap, but the control was not total on that scenarios. On the contrary, Grid is a multirow and multicolumn layout system since its definition

A container has to be declared with display: grid or inline-grid.

grid-template-columns, grid-template-rows and gap

The first one defines how many columns are part of the grid. Elements sits along created columns and creates as many rows as needed

The second one defines how many rows are part of the grid. If undefined, web browser implicitly creates as much as needed

the third one creates space between files and/or columns. But no space on the outher edges

Creating columns and rows with names and using a mixture of units

columns and rows creation is really flexible. Units can be mixed and names can be provided to refer to columns and rows by their names

grid-template-columns: [columna1] 150px [columna2] minmax(50px, 10%) [columna3] 1fr [columna4] 0.5fr [columna5] 1fr [fin-columnas];

grid-template-rows: [fila1] 25% [fila2] 2fr [fila3] auto [fin-filas]. It only can be noticed if container has a higher heigth than needed by its items. If no defined height in the container, the rows will be adjusted to the size of the content.

aligning containers and all items at once

justify-items, align-items and place-items

The first one allows to place items along the horizontal axis. values: start end center stretch

the second one allows to place items along the vertical axis. values: start end center stretch baseline. It just takes sense when vertical size of container is bigger than the addition of the size of its elements

the third one is a shortcut to place items along both axis at the same time

By default, grid elements occupy all the space in their container. Justify-content alters this behaviour and makes them occupy only what they need

aligning containers

justify-content, aling-content and place-content

the first one allows a grid container to be placed along the horizontal axis on its grid container. Values: start, end, center, stretch, space-around, space-between, space-evenly

the second one allows a grid container to be placed along the vertical axis on its grid container. Values: start, end, center, stretch, space-around, space-between, space-evenly

the third one is a shortcut for the previous ones

Important: it is only the container that moves. Its inner children are keeping their alignment at the inner container
